# dbxignore v0.4.0 — macOS support

dbxignore now runs on macOS — third platform after Windows and Linux. The project's marker-and-daemon architecture was built from the start with platform dispatch in mind, and v0.4 is the first release that actually exercises that flexibility against a third OS family.

## What's new for macOS users

dbxignore on macOS uses the `com.dropbox.ignored` extended attribute (the same key Dropbox itself reads) and registers as a launchd User Agent so the daemon auto-starts at GUI login.

```bash
pip install dbxignore
dbxignore install
```

Or grab pre-built arm64 binaries from this Release page and `xattr -d com.apple.quarantine` them to bypass Gatekeeper. Intel Mac users go via PyPI — the wheel is universal Python. Full instructions in the [macOS install section of the README](https://github.com/kiloscheffer/dbxignore#install-macos).

A few things specific to macOS:

- **Files split between two dirs.** State (`state.json`) lives in `~/Library/Application Support/dbxignore/`; logs (`daemon.log`, `launchd.log`) live in `~/Library/Logs/dbxignore/`. Matches Apple's app-data conventions.
- **Symlinks get marked on the link itself, not the target** — a small but meaningful behavioral improvement over Linux, where the kernel refuses xattrs on symlinks with `EPERM` and the daemon emits a WARNING. On macOS the marker lands silently and successfully.
- **First install needs a GUI login since boot.** `dbxignore install` bootstraps into the user's GUI domain; SSH-on-fresh-boot users will see `Bootstrap failed: 5: Input/output error`. Log into the GUI once, retry.

## What's new for Windows / Linux users

**No breaking changes.** The only user-visible delta is that the project description and README now mention macOS. State directories, command behavior, marker mechanism, daemon installation — all unchanged.

## Caveats

- **arm64 only for pre-built binaries.** Intel Mac users install via PyPI. If x86_64 demand surfaces, a point release will add an `macos-13` build leg.
- **Beta-validated, not field-validated at scale.** v0.4.0 was validated on real hardware by a beta tester before this release, but lacks the multi-user shake-out that Windows and Linux have accumulated. Please file issues for anything unexpected.
