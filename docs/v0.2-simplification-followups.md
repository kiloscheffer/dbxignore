# v0.2 simplification follow-ups

Code-level simplification candidates surfaced during the 2026-04-20 `/simplify` pass on the `feat/v0.1` branch. All tier-1 items (conftest extraction, `roots.find_containing`, daemon tidy-ups, in-place-rename reconcile skip) were applied in that pass. The items below were deliberately deferred — they're design-change territory, not pure simplification, and each deserves its own PR with focused tests.

## Deferred items

### 7. Skip reloading unchanged `.dropboxignore` files on sweep

**Where:** `daemon._sweep_once` calls `cache.load_root(r)` every hour, which re-reads and re-parses every `.dropboxignore` under the root regardless of whether watchdog events already kept the cache current.

**Why deferred:** needs mtime/size bookkeeping on the cache to decide "unchanged"; also raises a correctness question about what happens if watchdog missed an event (the sweep is the safety net).

**Shape of fix:** `RuleCache.load_root(root, force=False)` that stats each ignore file and skips unchanged ones; keep the full-reload path for startup or `force=True`.

## Non-deferrals (deliberately skipped as findings)

- **`cli.ads` vs `reconcile.ads` monkeypatch:** the `raising=False` in the pre-refactor `fake_ads` fixture was a no-op belt-and-suspenders — `cli` does import `ads` as a module attribute. The unified conftest fixture no longer needs it.
- **Parallel root reconcile:** independent roots could reconcile in parallel, but typical setups have 1–2 roots and Windows ADS writes serialize per-file anyway. Not worth the hazard.
